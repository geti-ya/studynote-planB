# 트랜잭션과 VTXO 기초

## 트랜잭션이란

비트코인 트랜잭션의 핵심 구성 요소:

1. **입력값 (Inputs):** 이전 족보 트랜잭션의 결과물을 정확히 참조하는 꼬리표
2. **결과물 (Outputs):** 트랜잭션 안에서 탄생한 새로운 주인의 몫 = **VTXO**
3. **위조 불가능한 서명 (Signatures):** 단순한 도장이 아니라, 알맞은 공개키와 암호화 값(sighashes)에 대해 찍힌 **MuSig2 통합 서명 및 개별 슈노르(Schnorr) 서명**. 배치를 만들 때 쓰이는 다중 서명(n-of-n MuSig2)도 여기에 포함됨
4. **탭루트 스크립트 트리 (Taproot Script Trees):** 돈을 쓰기 위한 세부 조건문. 탭 잎(tap leaves), 내부 키(internal keys), 머클 증명(Merkle proofs) 형태의 구조로 되어 있으며, 풀려면 알맞은 증거 데이터(witness data)를 제출해야 함
5. **시간 자물쇠 및 순서 제약 (Timelocks):** 사기꾼이 낡은 장부를 들이밀지 못하도록 시간을 통제. `nLockTime`, `nSequence`, `OP_CHECKLOCKTIMEVERIFY`, `OP_CHECKSEQUENCEVERIFY` 같은 명령어들
6. **해시 원상 조건 (Hash Preimage Conditions):** `OP_SHA256` 등의 해시 검증 조건. 주로 라이트닝 네트워크 등과 코인을 맞교환(Boltz submarine swap)할 때 사용

## VTXO란 — 영수증과 영수증 철

**VTXO (Virtual Transaction Output):** 비트코인 블록체인에 직접 올라가지 않은 '가상' 거래 출력값. "사전 서명된 미방송 비트코인 트랜잭션에 의해 뒷받침되는 오프체인 객체"다.

실질적인 돈(청구권)은 트랜잭션 안의 **아웃풋(Output)** 이다. 그런데 이 거래는 아직 비트코인 네트워크에 올라가지 않은 오프체인 상태여서, 이 결과물이 가짜가 아님을 증명하려면 영수증 전체가 필요하다.

- **VTXO (최신 영수증):** 방금 받은 딱 한 장의 가장 최신 트랜잭션. 가상 트랜잭션 트리의 '나뭇잎' 위치
- **족보 / DAG (장부):** 서버는 최신 영수증 한 장만 주지 않는다. 이 돈이 어디서부터 흘러왔는지 증명하기 위해, 과거 유저들끼리 주고받으며 형성된 **사전 서명된 트랜잭션들의 사슬 전체**를 함께 전달한다

→ VTXO를 받는다는 것 = **"최신 영수증 세트" + "과거 영수증 철(족보)"**을 함께 받는 것

족보 검증이란: 받은 거대한 영수증 철(DAG)을 펼쳐놓고, 내가 방금 받은 최신 영수증(VTXO 나뭇잎)부터 시작해 가장 밑바탕의 온체인 금고(배치 아웃풋 뿌리)까지 거슬러 올라가며 모든 과거 영수증들의 입력값과 결과물이 딱딱 맞물려 있는지 확인하는 것

## Outpoint란

**아웃포인트(Outpoint)** = txid(부모 트랜잭션 ID) + vout(몇 번째 결과물인지 나타내는 인덱스)를 하나로 묶어서 부르는 기술적 용어.

특정 트랜잭션의 특정 아웃풋을 정확히 가리키는 좌표값이다.
