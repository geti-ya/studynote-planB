# 아크 비상 탈출 완전 정리

## 비상 탈출(Unilateral Exit)이란

아크 서버가 먹통이 되어도 내 돈을 꺼낼 수 있는 구조.

근본적인 이유: **탭루트(Taproot) 주소의 스크립트 조건 안에 사용자가 돈을 되찾을 수 있는 규칙이 처음부터 프로그래밍되어 있기 때문이다.** 서버는 사용자의 탈출을 막을 수 없다.

금고 주소에는 처음부터 두 가지 지출 경로가 만들어진다:
- **경로 A** — 서버와 협력해서 자금을 움직이는 정상 경로
- **경로 B** — 서버 없이 혼자 빠져나오는 비상 경로

---

## 비상 탈출 실행 과정

1. **족보(트랜잭션 체인) 준비** — 오프체인에서 받은 사전 서명된 트랜잭션들의 체인을 들고 있다
2. **메인넷에 방송(Broadcast)** — 서버 없이 이 체인 전체를 비트코인 메인넷에 직접 제출
3. **언롤링(Unrolling) 및 온체인 확정** — 채굴자들이 족보를 검증하고 블록에 기록

오프체인 거래를 많이 할수록 트리 구조 깊은 곳에 위치 → 제출해야 할 트랜잭션이 길어진다 → 수수료 비쌈.

**편승(Jump off):** 나와 공통 경로를 공유하는 누군가가 이미 앞부분을 방송했다면, 나는 남은 짧은 부분만 제출하면 된다.

---

## 비상 탈출 규칙의 존재 증명 방법

"비상 탈출 가능합니다"라고 주장하는 게 아니라, **처음부터 금고 주소 안에 수학적으로 박혀 있어서 꺼낼 때 그 수학적 증거를 제시하면 된다.**

### 1단계 — 규칙은 어떤 언어로 쓰이는가

비트코인의 명령어(Script)로 작성된다.

```
<사용자 공개키>   OP_CHECKSIG             → "이 사람 서명이 맞는가?"
<특정 시간 조건>  OP_CHECKSEQUENCEVERIFY  → "지정된 시간이 지났는가?"
OP_DROP OP_TRUE                           → "두 조건 다 맞으면 통과"
```

### 2단계 — 두 규칙을 어떻게 하나의 주소에 숨기는가

```
경로 A 스크립트  →  SHA-256  →  Hash_A
                                        \
                                         SHA-256  →  Hash_AB ← 머클 루트
경로 B 스크립트  →  SHA-256  →  Hash_B  /
```

이 머클 루트를 공개키에 수학적으로 더해서(Tweak) 탭루트 주소를 만든다. 겉보기엔 평범한 주소지만 안에 두 경로가 해시로 박혀 있다.

### 3단계 — 탈출할 때 제출하는 3가지 데이터

| 항목 | 역할 |
|---|---|
| ① 경로 B 스크립트 원본 | "이게 내 비상 탈출 규칙입니다" |
| ② 내 디지털 서명 | "그 규칙의 조건을 내가 만족시킵니다" |
| ③ 컨트롤 블록 (Control Block) | "이 규칙이 처음부터 이 주소 안에 있었다는 수학적 증거" |

③ 컨트롤 블록에 짝꿍 **Hash_A**가 들어있다. "나는 경로 B를 공개하고, 짝꿍 Hash_A도 내놓을 테니 직접 계산해봐."

### 4단계 — 비트코인 노드가 검증하는 과정

```
① 경로 B 스크립트를 SHA-256  →  Hash_B 계산
② Hash_B + Hash_A 합쳐서 SHA-256  →  Hash_AB 계산
③ Hash_AB를 원래 금고 주소의 공개키에서 역산
④ 기존 주소와 일치하면 ✅  아니면 ❌
```

일치하면 노드는 "이 규칙이 주소가 만들어질 때부터 안에 있었던 게 맞네"라고 확인하고, 서명과 시간 조건을 최종 실행해서 돈을 내어준다.

## 왜 탭루트 규칙을 까발려야 하나 (핵심 Q&A)

**Q: 족보(사전 서명된 트랜잭션)만 바로 올리면 되는 거 아닌가?**

A: 탭루트 주소 안에는 경로 A (서버와 협력)와 경로 B (혼자 탈출) 두 가지가 있다. 메인넷은 기본적으로 경로 A를 기대한다. 아무 말 없이 족보를 제출하면 메인넷이 "이건 경로 A 규격이랑 다른데?"라며 거절한다.

탭루트 규칙을 까발리는 것은 서버에게 서명을 요구하는 게 아니라, 메인넷에게 **"나 경로 B 쓸 거야!"** 라고 선언하는 절차다.

**Q: 왜 전부 까발리지 않고 일부만?**

A: 탭루트의 머클 루트 덕분에 내가 사용하는 경로 B만 공개하면 된다. 경로 A는 공개할 필요가 없다 → 데이터 수수료 절약 + 프라이버시 보호.
