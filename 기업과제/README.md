# 기업과제 — ARK 프로토콜 VTXO 클라이언트 검증 구현

> Ark 프로토콜 기반의 비트코인 레이어2 SDK에서,  
> 서버를 맹목적으로 신뢰하지 않고 **클라이언트가 직접 VTXO를 검증**하는 기능을 구현한 기록.

### 1. 과제 설명

> 폴더: `과제 설명/`

| 파일 | 내용 |
|------|------|
| `과제_설명_완전정리.md` | Tier 1~3 요구사항 전체 정리 + 보고서 작성 가이드. 족보 재구성, 서명 검증, 온체인 앵커링, 스크립트 검증, 일방적 종료 구현까지 포함 |

**과제 3단계 구조 (Tier)**

- **Tier 1 (필수)** — VTXO 족보(DAG) 재구성 + 서명 검증 + 온체인 앵커 확인
- **Tier 2 (추가)** — 탭루트 스크립트 트리 분석, 타임락 검증, 해시 프리이미지 조건 검증
- **Tier 3 (고급)** — 서버 없이 일방적 종료(Unilateral Exit) 실행을 위한 데이터 로컬 저장 및 실행 함수 구현

## 핵심 개념 한 줄 요약

| 개념 | 설명 |
|------|------|
| VTXO | 아크에서 쓰는 오프체인 가상 코인 단위. '최신 영수증 1장' |
| 족보(DAG) | VTXO가 어디서 왔는지 증명하는 과거 트랜잭션 체인 전체. '영수증 철' |
| 커밋먼트 트랜잭션 | 족보의 가장 밑바닥, 온체인에 박혀있는 앵커. 세대가 바뀌면 값도 바뀜 |
| 탭루트 스크립트 | 돈을 꺼내는 조건들이 담긴 자물쇠. 내부 키 + 탭 잎 + 머클 증명으로 구성 |
| 비상 탈출 | 서버 없이 족보를 직접 메인넷에 방송해서 내 돈을 찾는 과정 |
| Zero Trust | 서버가 주는 값을 그대로 믿지 않고 직접 확인하는 설계 원칙 |
| TEE | ARK의 보안 가정. 하드웨어 신뢰 실행 환경. 단일 장애점(Single Point of Failure) |
