## 2. `src/index.ts` — SDK 정문(Entry Point)

`src/index.ts`는 이 SDK를 외부에서 사용할 때 쓸 수 있는 **모든 공개 클래스, 함수, 타입의 목록**이다. 이 파일만 봐도 SDK 전체 기능 범위를 파악할 수 있다.

### 핵심 export 카테고리

| 카테고리 | 주요 항목 |
|---|---|
| **지갑** | `Wallet`, `ReadonlyWallet`, `OnchainWallet`, `VtxoManager` |
| **프로바이더** | `RestArkProvider`, `RestIndexerProvider`, `EsploraProvider` |
| **스크립트** | `ArkAddress`, `DefaultVtxo`, `DelegateVtxo`, `VHTLC`, `VtxoScript` |
| **검증 유틸** | `verifyTapscriptSignatures`, `validateVtxoTxGraph`, `buildOffchainTx` |
| **트리** | `TxTree`, `TxTreeNode`, `SignerSession` |
| **타입** | `Vtxo`, `VtxoChain`, `ChainTx`, `CommitmentTx`, `Outpoint` |
| **네트워크** | `networks`, `Network`, `NetworkName` |
| **리포지토리** | `IndexedDBWalletRepository`, `InMemoryWalletRepository` |

내가 짠 `verifyVtxo` 함수에서 직접 쓴 것들:
- `TxTree`, `TxTreeNode` → vtxo 트리 구조 검증
- `validateVtxoTxGraph` → 서명 그래프 검증
- `ChainTx`, `VtxoChain`, `CommitmentTx` → 검증 대상 타입들
- `verifyTapscriptSignatures` → tapscript 서명 검증

## 6. `package.json` — 의존성 및 스크립트

프로젝트명: `@arkade-os/sdk` v0.4.14

### 핵심 의존성

| 패키지 | 용도 |
|---|---|
| `@scure/btc-signer` | 트랜잭션/PSBT 서명 (Transaction 클래스 여기서 옴) |
| `@noble/curves` | Schnorr 서명 수학 연산 |
| `@noble/secp256k1` | secp256k1 타원곡선 암호 |
| `@scure/base` | hex, base64 인코딩 |
| `@scure/bip39` | 니모닉(12단어 시드) 처리 |
| `bip68` | CSV(체크시퀀스버파이) 타임락 |

### 주요 테스트 명령어

```bash
pnpm test              # 전체 테스트
pnpm test:unit         # e2e 제외 유닛 테스트
pnpm test:integration  # e2e 테스트만
```

### 빌드 타겟

- `dist/esm/` — ES Module (브라우저/모던 JS)
- `dist/cjs/` — CommonJS (Node.js)
- `dist/types/` — TypeScript 타입 선언

Node.js 버전 요구사항: `>=22.12.0 <23`

## 7. 최상위 폴더 역할 요약 (코드 작업 관점)

과제를 처음 시작할 때 어느 폴더를 먼저 봐야 하는지 우선순위:

### 1순위: `test/` — 덫 놓는 곳

- 통합 테스트(Integration test) 코드들이 들어있음
- `test/e2e/` 안에 `wallet.test.ts`, `ark.test.ts` 등 실제 동작 예시가 있음
- VTXO 데이터를 `console.log`로 찍어보려면 여기서 먼저 실험
- 내 코드 작성 전에 "남들은 어떻게 썼나" 확인하는 커닝페이퍼 역할

### 2순위: `src/` — 진짜 코드 작성 현장

- SDK의 모든 실제 로직이 여기 있음
- `src/providers/indexer.ts`: `Vtxo`, `VtxoChain`, `ChainTx` 타입 정의
- `src/tree/validation.ts`: `validateVtxoTxGraph` 구현체
- `src/utils/arkTransaction.ts`: `verifyTapscriptSignatures` 구현체
- 테스트에서 데이터 구조 파악 후 이쪽에서 검증 함수 구현

### 3순위: 기술 문서 (`README.md`, `CONTRACTS.md`)

- **README.md** (39KB): 지갑 생성, 온보딩, 일방적 출금 등 전반적인 SDK 사용법
- **CONTRACTS.md** (3KB): ContractManager, Repositories 등 VTXO 데이터 흐름 아키텍처

> 힌트: `vitest.config.ts`가 보이면 → 테스트 실행 도구가 Vitest라는 뜻. `pnpm test`가 이 설정 기반으로 돌아감.
