# 기본 문법 정리 — const, 옵셔널 체이닝, async/await

## const — 한 번 정하면 바꿀 수 없는 이름표

```typescript
const tx = chain[i];
```

`const`는 **"이 이름이 가리키는 대상을 통째로 딴 걸로 바꾸지 않겠다"** 는 선언이다. 자물쇠처럼 잠긴다.

```typescript
// ❌ 통째로 갈아끼기 → 에러
tx = "hello";

// ✅ 상자는 그대로, 안의 속성만 바꾸기 → 통과
tx.status = "confirmed";
```

**왜 쓰나?** 코드가 수백 줄로 길어지면 중요한 데이터를 나도 모르게 덮어쓸 수 있다. `const`로 잠가두면 실수를 원천 차단한다. 값이 중간에 바뀌어야 하면 `let`을 쓴다.

## `?.` — 옵셔널 체이닝 (Optional Chaining)

```typescript
options?.vtxoTreeNodes
```

`options` 자체가 `null`이나 `undefined`여도 에러 없이 `undefined`를 반환한다. 안전하게 접근하는 방법.

```typescript
// options가 undefined이면 에러 없이 undefined 반환
const nodes = options?.vtxoTreeNodes;

// 여러 단계도 가능
const state = vtxo?.virtualStatus?.state;
```

## async / await — 비동기 함수

비동기란 **"작업이 끝날 때까지 프로그램 전체가 멈추지 않는다"** 는 뜻이다.

### 커피숍 비유

| 동기 | 비동기 |
|---|---|
| 커피 나올 때까지 계산대 앞에서 기다림 | 진동벨 받고 자리에서 책 읽다가 벨 울리면 픽업 |

### async 함수의 특징

1. `async` 함수는 **무조건 Promise를 반환**한다
2. 내부에서 오래 걸리는 작업을 만나도 프로그램 전체가 멈추지 않는다
3. `await`를 쓰면 그 줄에서만 결과를 기다린 뒤 다음으로 넘어간다

```typescript
async function fetchUser() {
    console.log("2. 요청 시작");
    const response = await fetch("https://api.example.com/user");  // 여기서만 기다림
    const data = await response.json();
    console.log("4. 완료:", data);
}

console.log("1. 시작");
fetchUser();
console.log("3. 다른 작업 진행 중...");

// 출력 순서: 1 → 2 → 3 → 4
```

`verifyVtxo`가 `async`인 이유: 온체인 조회(`verifyOnchainAnchoring`)가 비트코인 네트워크에 실제로 물어봐야 하는 작업이라 시간이 걸리기 때문이다.
