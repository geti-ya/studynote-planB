# verifyChainLinking — 체인 연결 검증 완전 정리

## 코드 원본

```typescript
export function verifyChainLinking(
    chain: ChainTx[],
    vtxoTxid: string
): { ok: boolean; error?: string } {

    if (chain.length === 0) {
        return { ok: false, error: "체인 데이터 없음" };
    }

    // chain은 leaf → root 순서
    // 각 tx의 spends 필드가 이전 tx를 올바르게 참조하는지 확인
    for (let i = 0; i < chain.length; i++) {
        const tx = chain[i];

        // leaf(첫 번째)는 우리의 VTXO를 포함해야 함
        if (i === 0) {
            if (!tx.txid.includes(vtxoTxid) && tx.txid !== vtxoTxid) {
                // arkTx의 경우 vtxo txid와 arkTxid가 다를 수 있음
                // spends 배열에서 우리 vtxo를 확인
            }
        }

        // COMMITMENT tx이면 이게 온체인 앵커 → 체인의 끝
        if (tx.type === ChainTxType.COMMITMENT) {
            if (i !== chain.length - 1) {
                return { ok: false, error: `commitment tx가 중간에 위치: index ${i}` };
            }
            break;
        }

        // 다음 tx가 현재 tx를 spends에 포함하는지 확인
        if (i < chain.length - 1) {
            const nextTx = chain[i + 1];
            const spends = nextTx.spends ?? [];
            if (!spends.includes(tx.txid)) {
                return {
                    ok: false,
                    error: `체인 연결 끊김: ${tx.txid}가 다음 tx의 spends에 없음`,
                };
            }
        }
    }

    return { ok: true };
}
```

---

## 함수 설명

이 함수는 **블록체인 트랜잭션 체인이 올바르게 연결되어 있는지 검증**하는 함수다.

### 체인의 구조
```
chain[0]        chain[1]        chain[2]
  LEAF    →       중간 TX   →    COMMITMENT
(VTXO 포함)               (온체인 앵커, 체인의 끝)
```

- `chain[0]` = leaf: 우리의 VTXO를 담고 있는 가장 처음 트랜잭션
- `chain[마지막]` = root: COMMITMENT 타입, 온체인에 실제로 기록된 앵커
- 각 트랜잭션은 **`spends` 필드**로 이전 트랜잭션을 참조해야 연결된다

### 단계별 흐름

**① 빈 배열 조기 반환** — 체인이 비어있으면 바로 실패

**② COMMITMENT 타입 검사** — COMMITMENT는 반드시 체인의 끝(root)이어야 한다. 중간에 있으면 오류

**③ 핵심: 체인 연결 검증**
```
chain[i].txid  →  반드시  →  chain[i+1].spends 안에 존재해야 함

  TX_A                          TX_B
  txid: "aaa"       ✅          spends: ["aaa", ...]  ← aaa 있음, OK!

  TX_A                          TX_B
  txid: "aaa"       ❌          spends: ["bbb", ...]  ← aaa 없음, 오류!
```

`??` 연산자: `nextTx.spends`가 `null`이나 `undefined`일 때 빈 배열 `[]`로 대체하는 null 병합 연산자

## 자꾸 에러났던 부분 — spends 포맷 문제

```bash
chain[1].spends 원본:  ["9ab3de303fded78010e81fa4d357dbcbc5c9854872af7a61f4d2e0c964faa675:0"]
chain[2].txid 원본: 9ab3de303fded78010e81fa4d357dbcbc5c9854872af7a61f4d2e0c964faa675
길이 비교: 66 64
```

`spends` 배열의 각 요소는 `"txid:vout"` 형식이었다. 단순히 txid만 들어있는 게 아니라 "txid의 몇 번째 자식인지"를 함께 표현하는 데이터였다.

**알기 전 코드 (틀림):**
```typescript
if (!spends.includes(nextTx.txid)) { ... }
```

**알고 나서 고친 코드 (맞음):**
```typescript
const isLinked = spends.some(spend => {
    const spendTxid = spend.split(":")[0];  // "txid:vout" → "txid"
    return spendTxid === nextTx.txid;
});
```

`spends`가 조부모 주소를 나타내는 게 아니라, 내가 어디서 온 자식인지를 가장 정확하게 표현해주는 데이터였다.
