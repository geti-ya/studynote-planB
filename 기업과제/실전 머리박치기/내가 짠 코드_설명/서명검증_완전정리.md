# verifyVtxoTreeSignatures — 서명 검증 완전 정리

## 코드 원본

```typescript
export function verifyVtxoTreeSignatures(
    vtxoTreeNodes: TxTreeNode[],
    commitmentTxHex: string,
    sweepTapTreeRoot: Uint8Array
): { ok: boolean; error?: string } {
    try {
        const graph = TxTree.create(vtxoTreeNodes);
        const commitmentTx = Transaction.fromPSBT(
            base64.decode(commitmentTxHex)
        );

        // SDK의 validateVtxoTxGraph 호출
        // → cosigner 공개키 집계 → taproot 스크립트 검증
        validateVtxoTxGraph(graph, commitmentTx, sweepTapTreeRoot);

        /*
        // 각 leaf(VTXO)에 대한 tapscript 서명 검증
        for (const leaf of graph.leaves()) {
            const result = verifyTapscriptSignatures(leaf);
            if (!result) {
                return { ok: false, error: `leaf 서명 검증 실패: ${leaf.id}` };
            }
        }
        */

        return { ok: true };
    } catch (err: any) {
        return { ok: false, error: `트리 검증 실패: ${err.message}` };
    }
}
```

---

## 코드 설명

이 함수는 3개의 파라미터를 받아 VTXO 트리 전체의 서명이 유효한지 검증한다.

| 파라미터 | 의미 |
| --- | --- |
| `vtxoTreeNodes` | VTXO들이 이진 트리로 구성된 노드 배열 |
| `commitmentTxHex` | 온체인에 기록된 커밋먼트 트랜잭션 (hex 문자열) |
| `sweepTapTreeRoot` | Taproot 스윕 트리의 루트 해시 (Uint8Array) |

### 1단계: 트리 구조 생성

```typescript
const graph = TxTree.create(vtxoTreeNodes);
```

배열 형태의 노드 데이터를 SDK의 `TxTree` 객체로 변환한다. 이 객체는 루트→리프 방향의 트리 관계를 내부적으로 관리한다.

### 2단계: 커밋먼트 트랜잭션 파싱

```typescript
const commitmentTx = Transaction.fromPSBT(
    base64.decode(commitmentTxHex)
);
```

hex 문자열로 받은 커밋먼트 트랜잭션을 base64 디코딩한 뒤 PSBT 포맷으로 파싱한다. 이 트랜잭션이 트리 검증의 기준점(온체인 앵커)이 된다.

### 3단계: SDK 검증 함수 호출

```typescript
validateVtxoTxGraph(graph, commitmentTx, sweepTapTreeRoot);
```

SDK 내장 함수가 다음을 순서대로 수행한다:
- cosigner 공개키들을 집계 (MuSig2)
- 각 노드의 taproot 스크립트 검증
- Schnorr 서명 유효성 확인

오류가 있으면 예외(`throw`)를 발생시킨다.

### 결과 반환

검증을 통과하면 `{ ok: true }` 반환. 예외가 발생하면 `catch`에서 잡아 `{ ok: false, error: err.message }` 반환.

주석 처리된 `for (const leaf of graph.leaves())` 블록은 리프별 tapscript 서명을 추가로 검증하는 코드인데, 현재는 `validateVtxoTxGraph`가 이를 포함하므로 비활성화 상태다.

## verifyVtxo에서의 호출 위치

이 함수는 `verifyVtxo`의 Check 3(서명 검증) 단계에서 호출된다. `verifyVtxo`는 호출 전에 세 파라미터가 모두 존재하는지 먼저 확인하고, 없을 경우 preconfirmed 여부로 분기한다.

```
vtxoTreeNodes + commitmentTxHex + sweepTapTreeRoot 모두 있음
    → verifyVtxoTreeSignatures() 호출 → { ok, error } 반환

셋 중 하나라도 없음
    → preconfirmed 상태면 signaturesValid = true 처리
    → 아니면 errors에 추가
```

`verifyVtxoTreeSignatures` 함수 자체는 세 파라미터가 모두 있다고 가정하고 검증만 수행한다. 파라미터 유무 체크는 호출자(`verifyVtxo`) 책임이다.
