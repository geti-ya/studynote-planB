# Node.js 환경 및 문제 해결

## Node.js란

Node.js는 '자바스크립트(또는 타입스크립트) 코드를 인터넷 브라우저 밖인 내 컴퓨터 자체에서 실행할 수 있게 해주는 프로그램(실행 환경)'이다. Arkade SDK 코드와 통합 테스트 환경이 모두 타입스크립트로 짜여 있기 때문에, 이 코드들을 컴퓨터에서 돌리려면 Node.js가 필수적으로 깔려 있어야 한다.

---

## 에러: pnpm이 인식되지 않음

```
pnpm : 'pnpm' 용어가 cmdlet, 함수, 스크립트 파일 또는 실행할 수 있는 프로그램 이름으로 인식되지 않습니다.
```

**원인:** 컴퓨터에 `pnpm`이 아직 설치되어 있지 않기 때문.

**해결 순서:**

1. Node.js가 설치되어 있다면, 파워쉘에 다음 명령어를 입력:
   ```
   npm install -g pnpm
   ```
2. 파워쉘 창을 완전히 닫았다가 다시 열기 (컴퓨터가 새로 설치된 pnpm을 인식하게 하기 위해)
3. ts-sdk 폴더로 이동 후 다시 시도:
   ```
   pnpm install
   ```

## 에러: REPL 밖에서 나가서 치세요

**상황:** 파워쉘 화면이 `>` 하나만 뜨는 상태

**원인:** 실수로 `node`라고 치고 엔터를 눌러서 일반 파워쉘을 벗어나 **Node.js 내부의 자바스크립트 대화창(REPL)** 안으로 들어가 버린 것이다. `npm`은 원래 터미널 환경에서 쓰는 명령어이기 때문에 여기서 치면 에러가 난다.

**해결:**
1. `Ctrl + D` 를 누르거나, `.exit` 를 타이핑하고 엔터
2. 화면이 `PS C:\Users\...\ts-sdk>` 모양으로 돌아왔는지 확인
3. 원래 치려던 명령어를 다시 입력

**만약 `.exit` 쳐도 창이 닫히는 경우:**
- 윈도우 시작 메뉴에서 'Node.js' 아이콘을 클릭해서 실행한 것이 원인이다
- 그 창을 완전히 닫고, 시작 메뉴에서 **PowerShell** 을 검색해서 실행하면 된다
