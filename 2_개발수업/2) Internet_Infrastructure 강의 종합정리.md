## 인터넷의 기원

### ARPANET

인터넷의 시초로, 1970년대 미국 정부 주도로 만들어진 군사 및 대학 연합 연구용 네트워크다.  
초기에는 UCLA, 스탠퍼드, MIT, 하버드 같은 주요 대학과 군사 시설들을 연결했으며, 영국과 노르웨이를 위성으로 연결한 것이 최초의 국제 연결이었다.  
1984년 군사망(MILNET)이 분리되어 순수 연구용 통신망으로 남은 것이 오늘날 인터넷의 모태가 되었다.

### TCP/IP 프로토콜

네트워크 상의 컴퓨터들이 데이터를 주고받기 위한 **세계 공통의 통신 규약**이다.  
1983년 1월 1일(Flag Day), 인터넷에 연결된 모든 장비가 일제히 TCP/IP로 전환되었다.  
데이터를 패킷 단위로 분할·주소 지정(IP)하고, 신뢰성 있는 전송 순서를 보장(TCP)하는 방식으로, 현재의 인터넷은 모두 이 TCP/IP 위에서 동작한다.

### 회선 교환 vs 패킷 교환

인터넷 이전의 구형 전화망은 **회선 교환(Circuit-switched)** 방식을 사용했다.

| 구분 | 회선 교환 | 패킷 교환 |
|------|-----------|-----------|
| **경로** | 통신 전 경로를 독점 예약, 끝까지 고정 | 경로 미고정. 패킷마다 Hop-by-hop으로 판단 |
| **효율** | 아무 말 없어도 회선 계속 점유 | 유휴 회선 낭비 없음 |
| **내결함성** | 중간 교환기 하나만 고장 나도 통신 두절 | 특정 구간 파괴돼도 패킷이 우회 경로를 스스로 탐색 |
| **패킷 크기** | 해당 없음 | 최대 1,500바이트 단위로 분할 전송 |

패킷 교환 방식은 **핵전쟁 같은 대규모 파괴를 견디기 위해** 설계되었다는 점이 핵심이다.

> 인터넷은 **데이터를 잘게 쪼개어 스스로 우회로를 찾게 만든 탈중앙화 생존형 통신망(ARPANET + TCP/IP)** 이다.

### 해저 케이블 vs 위성

대륙 간 장거리 통신은 위성이 아닌 총 길이 **140만 km에 달하는 해저 광케이블**을 통해 55~75ms의 빠른 속도로 이루어진다.  
스페이스X의 스타링크(Starlink)는 인터넷의 뼈대(백본, Backbone)가 아니라 말단 접속망(Leaf)에 불과하다.

## 네트워킹 기초

### 기본 장비

| 장비 | 계층 | 동작 방식 |
|------|------|-----------|
| **허브(Hub)** | 1계층 | 한 포트로 패킷을 받으면 연결된 **모든 포트**로 무조건 전송. 목적지 구분 없어 데이터 충돌(Collision) 발생. 현재는 거의 사라진 방식. |
| **스위치(Switch)** | 2계층 | '지능형 허브'. 목적지의 **MAC 주소**를 확인하고 해당 포트로만 정확히 전달. |
| **라우터(Router)** | 3계층 | 목적지의 **IP 주소와 서브넷**을 확인한 뒤, 가장 적합한 다음 라우터(Next hop)로 패킷 전달. |

### 정적 vs 동적 라우팅

- **정적 라우팅**: 관리자가 경로를 수동으로 입력. 홈 네트워크처럼 단순한 환경에 적합.
- **동적 라우팅**: 라우터들끼리 경로 정보를 자동으로 교환하고 업데이트.

전 세계 인터넷에는 **100만 개 이상의 네트워크 경로**가 존재한다. 사람이 수 밀리초(ms) 단위로 이를 수동 관리하는 것은 불가능하므로 동적 라우팅이 필수다.

### 자율 시스템(AS, Autonomous System)

단일 기술 관리 조직(ISP, 대기업, 대학 등)의 통제하에 운영되는 **독립적인 네트워크 또는 네트워크 그룹**이다.  
1개의 AS는 전 세계 인터넷을 구성하는 하나의 독립적인 참여자(노드) 역할을 한다.

- 내부 통신: **IGP(내부 게이트웨이 프로토콜)** 사용
- 외부 AS 간 통신: **EGP(외부 게이트웨이 프로토콜)** 사용
- **연속성 규칙**: 하나의 AS로 묶인 라우터들은 반드시 서로 연결되어 있어야 하며, 물리적/논리적으로 단절된 두 곳에서 같은 AS를 사용할 수 없다.

### ASN (Autonomous System Number)

각 AS를 식별하기 위해 부여되는 고유 번호로, 일종의 **우편번호** 같은 역할을 한다.

| 구분 | 내용 |
|------|------|
| **비트 체계** | 과거 16비트(1~65535) → 현재 32비트 표준 |
| **사설 ASN** | 64512~65534 (내부망·단일 업스트림 전용으로 예약) |
| **발급 기관** | RIPE(유럽/중동), ARIN(북미), APNIC(아태지역) 등 지역 인터넷 레지스트리(RIR) |

### IPv4 vs IPv6

- **IPv4**: 약 43억 개(2³²)의 주소 공간으로, 현재 거의 고갈된 상태
- **IPv6**: 2¹²⁸개의 사실상 무한한 주소 공간을 제공하며, IPv4 고갈 문제의 근본적인 해결책

### BGP (경계 경로 프로토콜, Border Gateway Protocol)

수많은 AS들을 서로 연결하여 오늘날의 거대한 인터넷망을 형성하는 **외부 동적 라우팅 프로토콜**이다.

**작동 원리**

BGP 라우터들은 **`TCP:179`** 를 통해 연결을 맺고, "나는 특정 목적지(IP 서브넷)로 가는 경로를 알고 있다"고 끊임없이 정보를 교환한다.  
연결은 Idle → Connect → Active → OpenSent → OpenConfirm → **Established** 순서로 수립되는 상태 머신(State Machine)으로 동작하며, 한번 Established 상태에 도달하면 끊어지지 않고 지속적으로 유지되도록 설계되어 있다.

**경로 선택 규칙**

```
① Longest Prefix Match (최우선)
   더 구체적인 서브넷이 AS-PATH 길이에 관계없이 항상 먼저 선택된다.
   예: /16 보다 /24 가 무조건 우선

② AS-PATH 길이 (동일 prefix일 때)
   목적지까지 거쳐야 하는 ASN 목록(AS-PATH)이 짧을수록 우선 선택된다.
```

**신뢰 기반 모델**

BGP는 근본적으로 **"검증 없는 신뢰(Trust, don't verify)"** 를 바탕으로 설계되었다.  
상대방 라우터가 특정 경로를 알고 있다고 선언하면, 시스템은 이를 그대로 믿고 트래픽을 보낸다.

## 심층 분석

### RIB vs FIB

| 구분 | 역할 |
|------|------|
| **RIB** (Routing Information Base) | BGP 라우터가 수신한 모든 경로를 저장하는 메모리 내 데이터베이스 |
| **FIB** (Forwarding Information Base) | RIB에서 선택된 최선의 경로만 담아, 실제 하드웨어/커널이 패킷을 전달할 때 참조하는 테이블 |

RIB에서 최선의 경로를 선택하여 FIB에 반영하고, 실제 패킷 전달은 FIB를 기준으로 동작한다.

### 트래픽 엔지니어링

- **나가는 트래픽(Outbound)**: AS-PATH 조작 등을 통해 비교적 쉽게 제어 가능
- **들어오는 트래픽(Inbound)**: 상대방 AS의 라우팅 결정에 의존하므로 통제하기 매우 어려움

## 신뢰와 보안

### BGP의 구조적 취약점

BGP의 '검증 없는 신뢰' 모델로 인해, 오설정이나 악의적 공격 한 건이 전 세계적인 인터넷 장애로 번질 수 있다.

### 실제 사고 사례

| 사건 | 내용 |
|------|------|
| **말레이시아 텔레콤** | 잘못된 BGP 설정으로 대규모 인터넷 경로 장애 발생 |
| **파키스탄 텔레콤** | 유튜브를 자국 내에서만 차단하려다 전 세계 유튜브를 다운시킴 |
| **페이스북** | 내부 설정 오류로 스스로 BGP 라우팅을 끊어 전 세계 접속 불가 |
| **MyEtherWallet 하이재킹** | BGP 하이재킹을 통한 악의적 공격으로 사용자 암호화폐 탈취 |

### 방어책

- **필터링 정책**: 비정상적인 BGP 경로 광고를 사전에 차단
- **RPKI** (Resource Public Key Infrastructure): 암호화 기반 경로 검증 시스템. IP 주소 블록의 정당한 소유자만이 해당 경로를 광고할 수 있도록 서명으로 검증

## 실제 구현

### 독립적인 네트워크 구축 (Self-hosting)

거대 클라우드에 의존하지 않고 직접 인터넷 인프라에 참여하는 방법이다.

| 수준 | 방법 |
|------|------|
| **홈랩** | 가정용 고정 IP를 사용해 소규모로 시작 |
| **ISP** | 전용 광케이블 임대 + 지역 레지스트리(RIPE 등) 가입 → 자신만의 ASN·IP 블록을 발급받아 실제 ISP로 참여 |

### 애니캐스트(Anycast) 라우팅

하나의 IP 주소를 여러 지리적 위치에서 동시에 서비스하는 라우팅 방식이다.  
사용자는 자동으로 가장 가까운(지연이 낮은) 서버로 연결된다.  
이를 비트코인 노드 운영에 접목하면, 단일 IP로 전 세계 여러 노드를 동시에 운영하는 분산 구조를 실험할 수 있다.

## 전체 흐름 요약

```
[내 컴퓨터]
    │
    ↓ 데이터를 패킷(최대 1,500B)으로 분할 — TCP/IP
라우터 (FIB 참조 → Next hop 결정)
    │
    │  AS 경계 통과 — BGP 피어링 (TCP:179)
    │  경로 선택:
    │  ① Longest Prefix Match (/24 > /16) ← 최우선
    │  ② AS-PATH 길이 (짧을수록 우선)
    ↓
[목적지 AS / 서버]

[백본 인프라]
    └─ 해저 광케이블 (140만 km, 55~75ms)
    └─ Anycast: 하나의 IP → 가장 가까운 노드로 자동 연결

[보안 레이어]
    └─ BGP 기본값: "검증 없는 신뢰" → 하이재킹·오설정 취약
    └─ RPKI: ROA 서명으로 IP 소유권 암호학적 검증
    └─ 필터링: 비정상 BGP 경로 광고 사전 차단
```

> **핵심**: 인터넷은 설계상 탈중앙화되어 있지만, 실제 경로 제어는 BGP를 통해 소수의 대형 ISP가 사실상 주도한다.  
> BGP의 구조적 취약점(`신뢰 기반 모델`)과 그 방어책(RPKI, 필터링)을 이해하는 것이 독립적인 인터넷 인프라 구축의 출발점이다.
